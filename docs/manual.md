# wechat-codex 命令使用手册

`wechat-codex` 是一个功能完备的命令行工具（CLI），专为管理和运行微信机器人轮询服务而设计。该程序拥有如同操作 Git 般开箱即用的子命令体验。

你可以随时通过 `bin/wechat-codex --help` 来查看主程序的总览与所有支持的命令。

以下是所有核心生命周期命令的详细说明：

---

## 1. 启动长连接轮询服务：`start`

**命令格式：**
```bash
bin/wechat-codex start [flags]
```

**功能介绍：**
该命令用于挂载监听引擎并启动长连接网络轮询服务。它会源源不断地接收从腾讯服务器下发的新微信消息（当前源码内实现了一个存根回调，支持自动回显文本消息）。

**支持的标志（Flags）：**
- `-d`, `--daemon`: 指示本进程以**后台守护进程**（Daemon）模式脱离控制台静默启动。

**场景分解：**

- **前台测试/开发模式**：
  ```bash
  bin/wechat-codex start
  ```
  引擎将直接在本地向标准输出（Stdout）打印所有的运行时状态与消息推送流日志。这非常适合你在进行二次开发或 Debug 测试时排查问题。你可以直接通过按 `Ctrl + C` 来安全终止退出。

- **后台生产托管模式**（推荐）：
  ```bash
  bin/wechat-codex start -d
  ```
  如果你要在云端服务器上挂机部署机器人，该命令将在启动后立刻交还终端控制权。你的 SSH 会话断开也不会导致机器人停止运行（会自动忽略 SIGHUP 信号）。
  - **运行日志记录**：会被后台托管并追加输出至 `~/.wechat-codex/service.log`。
  - **进程锁（PID）**：系统会自动记录当前后台状态，存放在 `~/.wechat-codex/wechat-codex.pid`。

---

## 2. 健康检查与探活：`status`

**命令格式：**
```bash
bin/wechat-codex status
```

**功能介绍：**
针对使用 `-d`（后台守护）模式启动的服务，你无法在终端直接看到该进程。此时使用该命令可以快速且无副作用地探活该服务的运行状态。

**工作原理：**
它会自动分析 `~/.wechat-codex/wechat-codex.pid` 中的进程号锁，并向系统内核发送存活嗅探，如果服务健康正常，将会打印出：“服务仍在运行，进程号为 [PID]”。你可以像巡检系统一样随时调用。

---

## 3. 停止与进程释放：`stop`

**命令格式：**
```bash
bin/wechat-codex stop
```

**功能介绍：**
如果你需要安全、优雅地结束一个脱离了终端而正在后台挂机的机器人服务，请立刻调用此命令。

**工作原理：**
你永远无需手动执行容易出现误杀风险的 `ps -ef | grep` 或 `kill -9`。系统会自动调取当前的锁环境文件，向目标守护服务发送正确的退出信号，并在停机完成后安全释放周边上下文清理环境。

---

## 💡 常见问题与 Tips

   所有的登录数据、状态位移书签和日志，全部都被物理隔离并隐藏缓存在全局应用配置文件夹 `~/.wechat-codex/` 等沙箱目录中。如果你希望彻底清除缓存、重置或更换机器人的微信号，可以暴力删除该目录然后重新执行 `start` 即可。

2. **探查隐藏特性**
   任何命令如果遗忘了用法，你可以通过附加 `-h` 标志请求指引。例如：
   `bin/wechat-codex start -h`
