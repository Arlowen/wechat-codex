# wechat-codex

![Go Version](https://img.shields.io/badge/Go-1.16+-00ADD8?style=flat&logo=go)
![CLI](https://img.shields.io/badge/CLI-Cobra-blue?style=flat)

`wechat-codex` 是一个基于 Go 语言编写的高性能微信机器人控制台与服务端基座。该项目深度对接腾讯 iLink (微信机器人平台) 官方 API，提供了一整套从**终端扫码身份授权**到**后台全自动长轮询接收及处理微信消息**的闭环服务体系。

本项目非常适合作为二次开发个人智能助手、自动回复机器人的开箱即用底层框架。

---

## 🏗 核心架构与底层设计

本项目在底层工程化设计上拥有以下技术亮点，为稳定运行提供保障：

1. **原生的消息长轮询 (Long-Polling) 引擎**
   在核心引擎 `wechat/service.go` 中，实现了稳定且鲁棒的 `CodexService.RunForever()`。它基于 iLink 的 `getupdates` 接口构建 35 秒超时级长连接轮询，并且具备完善的**异常恢复机制**：断网或 Token (error -14) 掉线时，能自动清理轮询位移 (`get_updates_buf`) 缓存并执行退避重试策略。

2. **状态与敏感凭证的安全隔离 (Local Store)**
   核心凭据不再零散分布于环境变量或根目录。通过 `wechat/store.go`，应用的所有敏感数据（如会话 `token`）及运行状态上下文（如每次请求下发的更新流位移进度），均被读写锁保证并发安全地序列化至工作目录下的专属隐藏缓存区 `.runtime/wechat/` 内，实现了服务平滑重启甚至迁移时不丢消息位移。

3. **完善的低门槛 API 客户端封装**
   专属的 API Client 极致解耦。无缝封装了标准请求头重写鉴权过程（自动注入 Bearer 和伪造客户端 UIN 设备指纹），不仅支持文本回复构建 (`sendmessage`)、还保障着终端登录流检活 (`get_qrcode_status`) 等核心网络交互安全。

4. **无感知的现代化守护进程 (Daemon) 托管**
   无需繁杂配置 `systemd` 或 `supervisor` 服务，也不依赖容易崩溃的 `nohup`，工具原生自带 `-d / --daemon` 标志，利用直接执行子进程 (`Fork`) 实现自托管守护运行。应用内自管理生成进程锁文件 (`wechat-codex.pid`) 并原生配置了运行日志文件流 (`service.log`) 。

## 🛠 极速构建

要求开发环境：**Go 1.16+**

你可以通过以下自带工具脚本一键完成静态编译并归档至 `bin/` 目录：
```bash
./all_build.sh
```

如果你只希望进行通用构建测试：
```bash
go build -o bin/wechat-codex .
```

## 📖 操作指南

程序入口是基于著名的 `spf13/cobra` 框架构建的优雅命令行，结构清晰，就像敲击 Git 命令一样顺畅：

### 1. 账号全自动扫码授权 (`login`)
**首次使用必填项**，如果没有有效的 `token` 执行其他命令将会报错退出：
```bash
bin/wechat-codex login
```
系统将会请求认证并通过 ASCII 画的方式在你的 Shell 终端直接打印绘制出认证二维码！拿出待授权的微信扫码即可完成登录授权，全部过程的 Token 将被系统智能保存托管。

### 2. 挂载轮询服务引擎 (`start`)
接收微信消息的核心监控引擎，支持两种模式：

- **前台活跃模式**（实时打印并查阅 DEBUG / INFO 日志流，推荐二次开发断点或测试阶段使用，`Ctrl+C` 退出）：
  ```bash
  bin/wechat-codex start
  ```

- **生产级后台托管运行模式**（静默巡航，断开 SSH / 终端也不会使服务掉线死亡）：
  ```bash
  bin/wechat-codex start -d
  # 当然，你也可以写完整参数 --daemon
  ```

*(注：由于本项目定义为开发基座，当前源码内实现了一个存根测试逻辑：当接收到用户文本消息时，机器人会自动回显一条「收到消息: <文本> (Go 版本 wechat-codex 存根回复)」)。当你进行实际开发时，可在该存根位置快速接入诸如 OpenAI / Claude 等流行的大语言模型以激活智能对话响应。)*

### 3. 可视化探针健康检查 (`status`)
如果你的机器人运行在后台托管模式下，你可以毫无副作用地调用以下指令探活：
```bash
bin/wechat-codex status
```
引擎将会从本地 `.runtime/wechat/` 沙盒读取进程快照并核实底层进程活动。

### 4. 释放与停止进程 (`stop`)
想要让机器人下线停止工作，无需再痛苦地用 `ps` + `grep` 检索再手动 `Kill`：
```bash
bin/wechat-codex stop
```
它内置了生命周期拦截，一键执行即可安全剔除进程。

## 💡 为什么需要本工具？

与繁重、臃肿或被官方重点监控的诸多 Python / Node 协议框架（如各种魔改的网页版微信协议）不同，通过 Go 语言编写的 `wechat-codex` 直接对接相对正规且轻量的 iLink 协议，同时具备被编译成无外部依赖**执行单文件**的绝对便携部署优势，并且具有比脚本型语言极出色的**常驻超低内存开销**及现代化的工程设计标准。你唯一需要考虑的，就是将其作为入口框架来构建专属于你的扩展功能。
